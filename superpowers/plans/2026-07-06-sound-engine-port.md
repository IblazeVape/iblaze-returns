# Sound & Haptics Engine Port Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port StarterCN's `useFeedback` sound/haptics engine (audio sound bank,
jotai-backed toggles, and `sound`/`sounds`-aware UI primitives) into
iblaze-returns, verbatim where possible, so that sub-project 3 (site chrome)
can consume the exact same props StarterCN's own components do.

**Architecture:** A pure-data sound bank (`audio/core.ts`) feeds a single
`useFeedback` hook that plays a sound (Web Audio, via `@web-kits/audio`) and
triggers a haptic pattern (via `web-haptics`) together, gated by two
jotai-backed boolean toggles. Four UI primitives (`button`, `popover`,
`dialog`, `drawer`) get thin wrappers around their existing `Root`/component
that call `useFeedback` on click or open/close state change.

**Tech Stack:** Next.js 15 / React 19 / TypeScript, `@web-kits/audio`,
`web-haptics`, `jotai`, `motion`, `next-themes`, `react-hotkeys-hook`
(all already installed except `jotai` and `motion`).

type KebabToCamel<K extends string> = K extends `${infer A}-${infer B}`
  ? `${A}${Capitalize<KebabToCamel<B>>}`

export type FeedbackType = {
  [K in PatchSoundKey]: KebabToCamel<K>;

const toFeedbackKey = (k: string): FeedbackType =>

const patchKeyByFeedback = Object.fromEntries(

    toFeedbackKey(k),

  ])
